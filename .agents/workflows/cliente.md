# Feedback de Cliente

Registra feedback, peticiones o incidencias reportadas por un cliente.

## Pasos

1. **Recopilar información**:
   - ¿Qué cliente? (nombre, instancia)
   - ¿Qué reporta? (bug, petición, mejora, queja)
   - Prioridad del cliente
   - Canal por el que llegó (email, llamada, chat)

2. **Clasificar**:
   - 🐛 Bug — Algo no funciona
   - ✨ Feature request — Quiere algo nuevo
   - 💡 Mejora — Algo funciona pero podría ser mejor
   - ❓ Pregunta — Necesita ayuda o documentación

3. **Registrar**:
   - Añadir a CONTEXT.md en sección apropiada
   - Si es bug urgente: añadir a 🔥 Urgente
   - Si es feature: evaluar prioridad y añadir a próximos pasos

4. **Definir acción**: ¿Se atiende ahora o se agenda?

## Notas

- El feedback de cliente siempre se registra, nunca se ignora
- Si afecta a varios clientes, subir prioridad
- Vincular con la instancia del cliente si es un bug específico
