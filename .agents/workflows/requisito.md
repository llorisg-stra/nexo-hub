# Nuevo Requisito

Documenta formalmente un nuevo requisito del proyecto.

## Pasos

1. **Recopilar información del requisito**:
   - Título descriptivo
   - Descripción detallada
   - Prioridad (🔥 Urgente / 🔴 Alta / 🟡 Media / 🟢 Baja)
   - ¿Es funcional o no funcional?
   - ¿Tiene dependencias?

2. **Formatear el requisito**:

   ```markdown
   | Campo | Valor |
   |-------|-------|
   | Título | [título] |
   | Prioridad | [prioridad] |
   | Tipo | Funcional / No funcional |
   | Dependencias | [lista] |
   | Descripción | [detalle] |
   ```

3. **Añadir a CONTEXT.md** en "Próximos Pasos" con la prioridad correcta

4. **Confirmar** con el usuario

## Notas

- Cada requisito debe ser atómico y verificable
- Si es complejo, descomponerlo en sub-tareas
